编译JDK源代码

发表于 2018-06-27 | 更新于 2025-11-14 | 分类于 java

重新编译JDK源代码，让其包含debug信息。

问题

当我们对程序进行debug时，有时候会追踪到jdk的源代码里面去。但是这时候会发现在IDE的debug界面上是无法查看类变量和本地变量信息（仅仅能按顺序查看方法参数的信息）。这是由于jdk自带的包rt.jar没有包含debug信息，所以需要重新编译源代码，并覆盖原有包rt.jar。
